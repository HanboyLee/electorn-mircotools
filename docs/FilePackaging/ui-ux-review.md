# 文件打包 · UI/UX 審查與改版方向

> 設計依據：`ui-ux-pro-max` Skill  
> 現狀代碼：`src/pages/FilePackaging/index.tsx`  
> 服務：`src/services/file-packaging`（掃描 / 打 ZIP / 打開路徑）  
> 產物：本文件 + `design-system.md` + `ui-flow.mermaid` + `ui-prototype.html` + `implementation-plan.md`

---

## 1. 功能與用戶目標

| 項 | 內容 |
|----|------|
| 頁面名稱 | 文件打包 |
| 核心任務 | 選本地目錄 → 掃描**同名多擴展名**文件組 → 打包為 ZIP → 在源目錄查看結果 |
| 默認擴展名 | `.ai` / `.eps` / `.jpg` / `.png`（與 ZipService 一致，**本期不改掃描規則**） |
| 用戶 | 設計師 / 素材整理桌面用戶（Electron） |
| 技術棧 | antd + styled-components；IPC 經 preload |

---

## 2. 現狀 UI 問題（對照 skill）

| 嚴重度 | 問題 | UX 規則 | 現狀 |
|--------|------|---------|------|
| **高** | 主操作混亂 | Hierarchy / CTA clarity | 「確認打包」「批量打包」、行內「打包」三者並存；**實際工作流只需「全部打包」** |
| **高** | 進度不可信 | Progress / Loading | `setInterval` 假進度到 90%，與真實 ZIP 無關 |
| **高** | 批量進度只顯示單條 | Progress Indicators | 多組打包時用戶不知第幾組 |
| **中** | 就緒原因不清 | Disabled clarity | 未選目錄時僅 emptyText；底欄式主 CTA 缺失 |
| **中** | 結果只在 Modal | Feedback | 關閉後難回顧；與「元數據寫入」摘要不一致 |
| **中** | 目錄信息弱 | Forms | 路徑純文本，無文件組數 / 擴展名說明徽章 |
| **中** | 表格 count 可能空白 | Data | `dataIndex: 'count'` 但掃描結果未必填 `count`（應用 `files.length`） |
| **低** | console.log 噪音 | — | 調試日誌 |
| **低** | 確認文案拗口 | Copy | 「是否將已選擇的路徑為以名稱對應的打包為zip包…」 |

**非 UI 但改版應對齊的產品事實**

- 打包歷史頁**已移除**（ZipService 註釋）；設計**不恢復**歷史中心。  
- 輸出固定在源目錄 `{組名}.zip`（無自選輸出路徑一期）。

## 3. 改版信息架構

```
┌──────────────────────────────────────────────────────────────────┐
│ 頁眉：標題「文件打包」+ 一句話說明                                 │
│      規則提示：同名 .ai/.eps/.jpg/.png → 各打成 {名}.zip           │
├──────────────────────────────────────────────────────────────────┤
│ 目錄卡片（工作目錄）：路徑 · 組數徽章                              │
│   [選擇目錄] [打開打包目錄]  ← 無「重新掃描」                      │
├───────────────────────────────┬──────────────────────────────────┤
│ 左：文件組一覽表（只讀）        │ 右：就緒檢查 + 本輪進度            │
│ 組名 / 擴展名 Tags / 文件數     │ 已選目錄 ✓  有文件組 ✓  可打包 ✓   │
│ （無行操作按鈕）                │ 打包中：i/m · 當前組               │
├───────────────────────────────┴──────────────────────────────────┤
│ 結果區：摘要 + 組結果列表（無打開目錄按鈕）                        │
├──────────────────────────────────────────────────────────────────┤
│ 底欄：次要說明「將打包全部 M 組」          主 CTA「全部打包」      │
└──────────────────────────────────────────────────────────────────┘
```

### 工作流定案（用戶反饋）

| 要 | 不要 |
|----|------|
| 選一個目錄、掃描出全部文件組 | 單組「打包」行按鈕 |
| **一次全部打包** 為唯一打包入口 | 勾選後「打包已選 / 批量打包」 |
| 表用於確認「會打哪些組」 | 用勾選決定打包範圍 |

> 一句話：**統一選目錄，統一全部打包。**

### 狀態機（不渲染 Steps）

| 階段 | 條件 | UI |
|------|------|-----|
| 空 | 無目錄 | 空態插畫文案 + 主按鈕「選擇目錄」 |
| 掃描中 | loading | 目錄卡 loading；表 skeleton/loading |
| 就緒 | 有組 | 表只讀一覽；底欄「全部打包」 |
| 無匹配 | 有目錄但 0 組 | 空結果說明擴展名規則 + 再選目錄 |
| 打包中 | isPackaging | 底欄 loading；進度 i/m；禁選目錄與再次打包 |
| 結果 | 有 ZipResult[] | 結果面板 + 可再打包 |

## 4. 關鍵界面元素

### 4.1 目錄卡片（工作目錄）

- 路徑全文 + 省略 tooltip  
- Tag：`{n} 個文件組`  
- 次要說明：僅掃描 `.ai .eps .jpg .png`  
- 操作（從左到右）：  
  1. **選擇目錄**（選完即掃描；換目錄 / 想刷新列表都靠再選一次）  
  2. **打開打包目錄**（**唯一**打開目錄入口；ZIP 與源文件同目錄）  
- **不做「重新掃描」**（用戶反饋：實際幾乎不用，再選目錄即可）  
- 無目錄時「打開打包目錄」disabled + tooltip「請先選擇目錄」  
- 打包中：建議禁用「選擇目錄」，「打開打包目錄」仍可用  
### 4.2 就緒檢查

| 項 | 就緒條件 |
|----|----------|
| 已選目錄 | `selectedDirectory` 非空 |
| 掃描完成 | 非 loading |
| 可打包 | `fileGroups.length > 0` |

打包中顯示真實進度，不用假百分比動畫作唯一信息。

### 4.3 文件組表（始終表格 · 只讀一覽）

| 列 | 內容 |
|----|------|
| 組名 | `name` |
| 擴展名 | Tags：`.ai` `.jpg`…（一眼看出該組含哪些格式） |
| 文件數 | `files.length` |
| 大小 | 可選：合計 size 格式化 |
| 狀態 | 待打包 / 排隊 / 打包中 / 成功 / 失敗（打包過程與結果回寫） |

- **無勾選列**、**無行內打包**、**無「詳情」按鈕**（用戶反饋：看不出詳情用途；擴展名列 + 文件數已足夠掃讀）  
- 搜索：按組名 / 擴展名過濾（僅影響展示；打包仍針對全部組）  
- 分頁：默認 20 或 50  
- `rowKey`：`id` 優先，否則 `name + basePath`  

### 4.4 主操作（唯一打包入口）

| 操作 | 位置 | 行為 |
|------|------|------|
| **全部打包** | 底欄主 CTA（唯一 `type=primary` 打包動作） | 打包當前掃描結果中的**全部**文件組 |
| **選擇目錄** | **工作目錄**卡 | 彈框選路徑並掃描（含「刷新」語義） |
| **打開打包目錄** | **工作目錄**卡（唯一） | 打開 `selectedDirectory`；**不在**結果區重複 |

**明確刪除（相對現狀與舊稿）**

- 頂部「確認打包」「批量打包」並排  
- 底欄「打包已選」  
- 行內「打包」  
- 行內「詳情」  
- 為打包而存在的多選 checkbox  
- 結果列表**每一行**的「打開位置」  
- 結果區再放一個「打開打包目錄」（與工作目錄重複）  
- **重新掃描**按鈕（再選目錄即可）  

確認 Modal（全部打包前，建議保留）：

> 將在 `{dir}` 下為 **全部 {N} 個**文件組各生成一個 ZIP（同名 ZIP 可能被覆蓋）。是否繼續？

文案用通順繁體，避免現有拗口句。

### 4.5 真實進度

```
正在打包 3/12 · logo-set
[████████░░░░] 25%
```

- `percent = completedGroups / totalGroups`  
- 單組內若 IPC 無細進度，組內可 indeterminate 或完成跳變；**禁止**無依據的 +10% 定時器  
- 結束：`成功 s · 失敗 f`

### 4.6 結果區（用戶反饋修訂）

**問題 1**：原「詳情」用途不清 → **不做**行級詳情入口。  
**問題 2**：原每個 ZIP 一行「打開位置」易誤解為多個目錄 → ZIP 均寫入**同一源目錄**，只應有 **一個**「打開打包目錄」。

#### 結構

```
┌─ 工作目錄 ──────────────────────────────────────┐
│ 路徑 · 組數                                      │
│ [選擇目錄] [打開打包目錄]                         │
└──────────────────────────────────────────────────┘

┌─ 打包結果 ──────────────────────────────────────┐
│ 摘要：合計 / 成功 / 失敗 / 成功率                 │
│ 列表（掃讀用，無打開按鈕）：                      │
│  hero-banner.zip   成功 · 3 個文件               │
│  logo-set.zip      成功 · 3 個文件               │
│  card-front.zip    失敗 · 文件被佔用…            │
│ 提示文案（可選）：ZIP 已寫入上方「工作目錄」      │
└──────────────────────────────────────────────────┘
```

| 元素 | 規則 |
|------|------|
| 打開打包目錄 | **只放在工作目錄區塊**；`ZipIPC.OPEN_ITEM(selectedDirectory)` |
| 結果列表 | 組名.zip + 成功/失敗 + 可選文件數 / 失敗原因；**無**行級打開、**無**區塊級打開 |
| 結果區文案 | 可一句「ZIP 已輸出至工作目錄」，引導用戶用上方按鈕 |

結果默認**內嵌面板**；不必只靠 Modal。

### 4.7 空態

**未選目錄**

- 標題：開始文件打包  
- 說明：選擇含同名多格式素材的文件夾  
- CTA：選擇目錄  

**有目錄但 0 組**

- 說明：未找到可分組文件（需同名且擴展名為 ai/eps/jpg/png）  
- CTA：選擇目錄（更換或重選同一路徑以刷新）

## 5. 交互規格

| 場景 | 行為 |
|------|------|
| 打包中選目錄 | 禁用 |
| 打包中 | 禁用「全部打包」與「選擇目錄」；「打開打包目錄」可用；表僅更新狀態列 |
| 覆蓋已有 zip | 維持現服務行為（archiver 寫同名）；UI 可在確認框提示「同名 ZIP 將被覆蓋」 |
| 打開打包目錄失敗 | message.error |
| 搜索清空 | 恢復全表展示 |

## 7. 非目標（本期）

- 不恢復打包歷史頁  
- **不做「重新掃描」**（再選目錄即可）  
- 不改掃描擴展名集合（若要可配置，另開需求）  
- 不自選輸出目錄 / 壓縮等級 UI  
- 不引入新組件庫或 Skill 深色玻璃皮膚  
- 不做相簿式網格牆（表格式工具）  

---

## 8. 驗收標準

1. 未選目錄時空態與主操作清晰。  
2. **唯一**打包入口「全部打包」；無勾選打包、無單組打包。  
3. 打包進度為組級 n/total + 當前組名，無假進度主導。  
4. 結束後結果摘要可掃讀；「打開打包目錄」在**工作目錄**區塊且全局唯一，結果區不重複。  
5. 表格文件數正確（`files.length`）。  
6. 視覺與其他改版頁 token 一致。
